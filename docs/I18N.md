# Internationalization

i18n status: N/A

## Reason

outcome-receipts v0.1 ships English-only. Report narrative templates, CLI
messages, and the trace view (`trace.html`) are not yet externalized into a
resource bundle; every user-facing string is hard-coded English in
`src/outcome_receipts/`.

## Reconciling with CLAUDE.md

[`CLAUDE.md`](../CLAUDE.md) states the eventual bar under Standards Conformance:
"Report output and any reviewer-facing copy are externalized; EN and ES at
parity for public-facing strings." That is the target for the v0.4+ mapping
work (once the tool is used by more than one org and translation has a
concrete audience), not a claim about v0.1's current state. Branch
`roadmap/e9-en-es-report-output-parity` tracks the externalization work; it is
unmerged as of this writing.

Until that lands, N/A is the accurate declaration for v0.1, and this file is
the committed record of that decision (per DOCUMENTATION-STANDARD's
N/A-with-reason requirement). Re-evaluate this file's status when
report-narrative templates or the trace view gain a second language.
