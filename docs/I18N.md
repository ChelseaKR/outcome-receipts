# Internationalization

i18n status: Applies to public report artifacts; English and Spanish are at parity.

## Supported surface

`receipts run --locale en|es` localizes reviewer-facing output while preserving
the underlying evidence. The report, approval language, provenance statement,
comparison and reconciliation labels, chart alternative text, receipt metadata,
and trace view are available in English and Spanish.

The following remain byte-identical across locales:

- computed figure values and canonical displays;
- SQL and metric definitions supplied by the report author;
- row counts, slice hashes, timestamps, and artifact semantics;
- receipts used by verification and bundle sealing.

Operational CLI progress and error messages remain English. Author-provided
report templates, titles, metric descriptions, definitions, caveats, and source
category values are not machine-translated; an organization supplies those in
the desired language. The optional Bedrock drafter does not provide a translation
guarantee.

## Verification

The externalized copy catalog lives in `src/outcome_receipts/copy.py`. Tests pin
EN/ES catalog completeness, localized grounding, approval and reconciliation
copy, chart accessibility text, and byte-identical figure evidence.
